# Руководство пользователя: Администратор (HQ)

Это руководство для ролей `ADMIN`, `MANAGER` и `SALES_MANAGER`, которые работают в кабинете HQ: `/admin`.

## 1. Зона ответственности

Администратор отвечает за:
- обработку интернет-заказов и контроль очереди продаж;
- приемку партий от франчайзи;
- распределение товаров по каналам продаж;
- управление контентом (локации, товары, переводы);
- управление пользователями и доступами;
- контроль операционных показателей.

## 2. Вход в систему

1. Откройте `/admin/login`.
2. Введите email/пароль учетной записи администратора, менеджера HQ или менеджера продаж.
3. После входа система автоматически откроет `/admin`.

Если после входа вы попали не в админку:
- проверьте, что у учетной записи роль `ADMIN`, `MANAGER` или `SALES_MANAGER`;
- обратитесь к владельцу системы или другому админу для смены роли.

## 3. Разделы админ-кабинета

- `Dashboard` — сводка по операционным метрикам.
- `Orders` — очередь заказов с сайта.
- `Acceptance` — приемка товаров по партиям.
- `Allocation` — распределение товаров на каналы продаж.
- `Страница клона` — live-редактирование текста публичной страницы цифрового клона.
- `Locations` — управление локациями на глобусе.
- `Products` — управление товарами.
- `Users` — управление пользователями и ролями.

## 4. Ежедневный рабочий цикл

## 4.1 Начало смены (5-10 минут)

1. Откройте `Dashboard`.
2. Проверьте:
   - `Batches in Transit` (что требует приемки);
   - `Items in HQ Stock` (что готово к распределению);
   - `Users/Franchisees` (нет ли отклонений по доступам).
3. Перейдите в `Acceptance`, если есть партии в пути.

## 4.2 Приемка партии (`Acceptance`)

Цель: перевести товары из `NEW` в `STOCK_HQ` или `REJECTED`.

Шаги:
1. Выберите партию из списка `Transit batches` или введите ID вручную.
2. Сканируйте/введите `temp_id`.
3. Сверьте фото и фактический товар.
4. Нажмите:
   - `Accept` — если товар принят;
   - `Reject` — если товар бракованный/не соответствует.
5. Когда не осталось `NEW`, нажмите `Finish Batch`.

Правило:
- кнопку `Finish Batch` нажимайте только после обработки всех позиций.

## 4.3 Заказы сайта (`Orders`)

Цель: быстро находить заявки покупателей, вести по ним рабочую карточку и закрывать после обработки.

Шаги:
1. Откройте `Orders`.
2. При необходимости используйте поиск по:
   - `order id`;
   - логину или имени покупателя;
   - телефону, email;
   - адресу доставки.
3. Выберите фильтр:
   - `Активные` — все `NEW` и `IN_PROGRESS`;
   - `Новые` — входящий поток;
   - `В работе` — заявки, которые уже взял менеджер;
   - `Закрытые` — `COMPLETED` и `CANCELLED`.
4. В левой колонке выберите заявку и откройте рабочую карточку справа.
5. В карточке заказа проверьте:
   - логин/имя покупателя;
   - телефон, email, адрес;
   - комментарий;
   - внутреннюю заметку менеджера;
   - состав заказа и сумму.
6. Для изменения данных нажмите `Редактировать`, затем используйте `Сохранить` или `Отменить`.
7. Учитывайте правило по закрытым заказам:
   - телефон, email, адрес и комментарий доступны только для чтения;
   - внутренняя заметка остается редактируемой.
8. Используйте только допустимые переходы:
   - `Взять в работу` для `NEW`;
   - `Закрыть` для `IN_PROGRESS`;
   - `Отменить` для `NEW` или `IN_PROGRESS`.

## 4.4 Распределение товаров (`Allocation`)

Цель: перевести товары из `STOCK_HQ` в канал продаж.

Шаги:
1. Откройте `Allocation`.
2. При необходимости используйте поиск по `temp_id`/`item id`.
3. Выберите товары:
   - вручную;
   - `Select Visible` для массового выбора.
4. Выберите канал:
   - `Online Marketplace` — товар уходит в онлайн;
   - `Offline Consignment` — выберите конкретного франчайзи и отправьте ему.
5. Убедитесь, что операция прошла без ошибки.

## 4.5 Контент и каталог (`Locations`, `Products`)

`Locations`:
- создавайте/редактируйте локации;
- загружайте изображения;
- поддерживайте переводы.

`Products`:
- создавайте/редактируйте карточки товаров;
- проверяйте цену, категорию, локацию;
- указывайте ссылки на Wildberries и Ozon для внешних кнопок заказа;
- добавляйте переводы.

Практика качества:
- не оставляйте карточки без изображения;
- проверяйте, что базовый язык заполнен перед переводами.

## 4.6 Пользователи (`Users`)

Цель: поддерживать актуальные роли и доступы.

Шаги:
1. Откройте `Users`.
2. Для создания пользователя нажмите `Add User`.
3. Заполните: имя, email, пароль, роль.
4. После создания нажмите `Refresh`.

Рекомендации:
- роль `ADMIN` выдавайте ограниченному кругу;
- для операционной работы используйте `MANAGER`;
- для сайта и входящих заказов используйте `SALES_MANAGER`;
- партнеров создавайте как `FRANCHISEE`.

## 4.7 Контроль QR-потока франчайзи

Администратор/менеджер может проверять QR-пакет любой партии:
- через API `GET /api/batches/:batchId/qr-pack`;
- через доступ к данным позиции (`clone_url`, `qr_url`) для диагностики ошибок печати или ссылок.

Что проверять при обращении франчайзи:
1. Партия существует и принадлежит корректному владельцу.
2. В `items[]` есть `clone_url` и `qr_url` для каждой позиции.
3. QR endpoint (`/api/public/items/:publicToken/qr`) отдает PNG без ошибок.
4. Публичная страница (`/clone/:publicToken`) открывается и показывает фото/видео.

## 5. Контрольные чек-листы

## 5.1 Чек-лист конца дня

- Все партии `TRANSIT`, пришедшие сегодня, обработаны или зафиксирован план на завтра.
- Нет товаров, зависших в `NEW` без причины.
- Товары `STOCK_HQ` распределены по приоритету продаж.
- Новые пользователи созданы с корректными ролями.
- Ошибки в интерфейсе зафиксированы и переданы в техподдержку.

## 5.2 Еженедельный чек-лист

- Сверка количества партий: отправленные/принятые/завершенные.
- Контроль зависших `STOCK_HQ`.
- Проверка качества данных каталога (изображения, переводы, цены).
- Аудит прав доступа пользователей.

## 6. Типовые проблемы и действия

## 6.1 Партия не находится в приемке

Проверьте:
- что партия уже отправлена франчайзи (`TRANSIT`);
- правильность ID;
- обновите список `Transit batches`.

## 6.2 Не получается распределить товар

Проверьте:
- статус товара должен быть `STOCK_HQ`;
- для `Offline Consignment` обязательно выбран франчайзи;
- повторите действие после `Refresh`.

## 6.3 Ошибка при создании пользователя

Проверьте:
- email не должен дублироваться;
- пароль достаточной длины;
- корректность роли.

## 7. Правила безопасности

- Не передавайте учетные данные другим сотрудникам.
- Не используйте общий admin-аккаунт для нескольких людей.
- Выходите из системы по `Sign Out` при завершении работы.
