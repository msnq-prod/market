# Руководство пользователя: HQ

Документ предназначен для ролей `ADMIN`, `MANAGER` и `SALES_MANAGER`, работающих в админ-контуре `/admin`.

## 1. Кто что делает

### `ADMIN`

- полный доступ к HQ-интерфейсам;
- приемка, склад, каталог, пользователи, контент;
- может вручную править ограниченный набор полей `Item`.

### `MANAGER`

- приемка, склад, каталог, пользователи;
- не работает с очередью заказов сайта.

### `SALES_MANAGER`

- работает только с `/admin/orders`.

## 2. Вход

1. Откройте `/admin/login`.
2. Введите email и пароль.
3. После входа:
   - `ADMIN` и `MANAGER` попадают в HQ-кабинет;
   - `SALES_MANAGER` попадает сразу в `/admin/orders`.

## 3. Разделы HQ

- `Dashboard`
- `Orders`
- `Acceptance`
- `Allocation`
- `Warehouse`
- `Locations`
- `Products`
- `Users`
- `Страница клона`
- `Video Tool`
- `QR Print`

## 4. Ежедневный рабочий цикл HQ

## 4.1 Начало дня

1. Откройте `Dashboard`.
2. Проверьте:
   - партии в `TRANSIT`;
   - партии в `RECEIVED`;
   - item в `STOCK_HQ`;
   - item в `STOCK_ONLINE`.
3. Если есть партии в пути или полученные, перейдите в `Acceptance`.

## 4.2 Приемка партии

### Шаг 1. Получить партию

1. Откройте `Acceptance`.
2. Выберите партию в `TRANSIT`.
3. Нажмите действие получения партии.

Результат:

- batch перейдет в `RECEIVED`.

### Шаг 2. Проверить позиции

Для каждой позиции:

- проверьте item внутри партии;
- примите его или отклоните.

Результат:

- `accept` -> `STOCK_HQ`
- `reject` -> `REJECTED`

### Шаг 3. Подготовить media

После приемки партии:

- загрузите или сопоставьте фото по `serial_number`;
- откройте `Video Tool`, если нужно собрать итоговые видео;
- убедитесь, что у каждого item есть и фото, и видео.

### Шаг 4. Напечатать QR

При необходимости:

- печать доступна из HQ-контура;
- используйте печать выбранных или всех публичных QR.

### Шаг 5. Завершить партию

Финализируйте batch только если:

- batch уже в `RECEIVED`;
- нет активного video processing / export;
- у каждого item есть `item_photo_url` и `item_video_url`.

Результат:

- batch -> `FINISHED`
- связанные `NEW` item -> `STOCK_HQ`

## 4.3 Заказы покупателей

Доступно:

- `ADMIN`
- `SALES_MANAGER`

Что можно делать:

- искать заказы;
- открывать карточку заявки;
- редактировать данные клиента у незакрытых заказов;
- вести внутреннюю заметку;
- менять статус.

Допустимые переходы:

- `NEW -> IN_PROGRESS`
- `NEW -> CANCELLED`
- `IN_PROGRESS -> COMPLETED`
- `IN_PROGRESS -> CANCELLED`

Важно:

- `internal_note` не виден покупателю;
- закрытые заказы не позволяют менять клиентские поля.

## 4.4 Allocation

Используется для item в `STOCK_HQ`.

Текущий рабочий сценарий:

- выбрать позиции;
- отправить их в онлайн-канал;
- получить `STOCK_ONLINE`.

В текущем UI:

- доступна только онлайн-алокация.

## 4.5 Warehouse

Используйте склад для:

- поиска партий и item;
- просмотра `serial_number`, `temp_id`, media и статусов;
- контроля clone URL и QR;
- просмотра итогового состояния по локациям и товарам.

Ручная правка item:

- только для `ADMIN`
- только для ограниченных полей

## 4.6 Locations

Что делать:

- создавать локации;
- редактировать координаты, изображение и базовый текст;
- поддерживать переводы.

Нельзя:

- удалять локацию, если к ней уже привязаны товары.

## 4.7 Products

Что делать:

- создавать товарные шаблоны;
- редактировать цену, фото, коды и тексты;
- публиковать или снимать с публикации;
- задавать ссылки внешних площадок;
- создавать задачу на сбор.

Нельзя:

- удалять шаблон, если он уже использован в партиях, задачах на сбор или заказах.

## 4.8 Users

Используйте раздел для:

- создания staff-аккаунтов;
- создания франчайзи;
- контроля ролей и балансов.

Ограничения:

- `MANAGER` не может создать `ADMIN`;
- `MANAGER` не может создать нового `MANAGER`.

## 4.9 Страница клона

Раздел `/admin/clone-content` нужен для:

- редактирования текстов цифрового паспорта;
- проверки live preview;
- проверки страницы по реальному `serial_number`.

## 5. Проверки качества

## 5.1 В конце дня

- нет партий, зависших в `TRANSIT` без движения;
- все полученные партии либо в работе, либо завершены;
- нет item в `NEW` без понятной причины;
- item из `STOCK_HQ` распределяются без накопления;
- новые пользователи созданы с корректными ролями.

## 5.2 Раз в неделю

- аудит ролей;
- проверка каталога;
- проверка качества публичных страниц и QR;
- сверка партий `TRANSIT -> RECEIVED -> FINISHED`.

## 6. Типовые проблемы

### Партия не переходит в `RECEIVED`

Проверьте:

- что она действительно в `TRANSIT`;
- что у пользователя есть роль HQ staff.

### Партия не финализируется

Проверьте:

- нет ли активного video-процесса;
- у всех ли item есть фото и видео;
- batch уже в `RECEIVED`.

### Не получается распределить item

Проверьте:

- item находится в `STOCK_HQ`;
- операция не выполняется повторно.

### QR недоступен

Проверьте:

- item не `REJECTED`;
- batch в `RECEIVED` или `FINISHED`;
- у item есть актуальный `serial_number`.

## 7. Правила безопасности

- не используйте общий admin-аккаунт на несколько сотрудников;
- не передавайте пароли другим людям;
- после работы выходите из системы;
- права `ADMIN` выдавайте минимальному числу сотрудников.
