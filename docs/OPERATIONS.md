# Эксплуатация

## Окружения и БД
- База по умолчанию: MySQL 8.0+ (InnoDB, utf8mb4)
- dev: локальная разработка
- staging: предпрод с авто-миграциями
- prod: боевое окружение

## Миграции
- Все изменения схемы только через `prisma migrate`
- Миграции прогоняются на staging до prod
- Запрещено делать ручные DDL-правки в prod

## Резервное копирование
- Ежедневные бэкапы базы данных
- Регулярная проверка восстановления (минимум раз в квартал)

## Мониторинг
- Включить slow query log в MySQL
- Собирать p95 и p99 по ключевым эндпоинтам
- Алертинг на рост ошибок и деградацию ответа

## Производительность
- Использовать read-replicas при росте нагрузки на чтение
- Кэшировать справочные данные в Redis
- Следить за количеством соединений и пулом
